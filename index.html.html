Math

#

Signature

Purpose

1

abs(x : int, float)

-> absolute value of a number : int

2

sum(iterable, start=0)

-> Sums items of an iterable

- start is inclusive - use math.fsum() for floats

3

divmod(numerator, denominator)

-> (quotient, remainder)

- For int -> (a // b, a % b)
- For float -> (q, a % b) : q <= math.floor(a / b)

- remainder takes sign of denominator

4

max(iterable, *[, key, default])

max(arg1, arg2, *args[, key])

-> largest item in an iterable

- key=ordering_function

- if not iterable: default=ret else ValueError

- If multiple max -> first occurence

5

min(iterable, *[, key, default])

min(arg1, arg2, *args[, key])

-> smallest item in an iterable

- key=ordering_function

- if not iterable: default=ret else ValueError

- If multiple mins -> first occurence

6

pow(base, exp[, mod])

-> base ** exp if pow(base, exp)

-> pow(base, exp) % mod) if pow(base, exp, mod=x)

- supports: int, float

7

round(number[, ndigits])

-> number rounded to ndigits after decimal point

- if ndigits is omitted or None -> nearest integer

- if .5 rounding is done toward the even number

8

bin(x : int or x.__index__())

-> converts to binary string prefixed with ‘0b’

- ‘-0b’ for negatives

- f'{14:#b}', f'{14:b}' -> ('0b1110', '1110')

9

hex(x : int or x.__index__())

-> converts to lowercase hexadecimal str, prefix ‘0x’

- f'{255:#x}', f'{255:x}', f'{255:X}'

-> ('0xff', 'ff', 'FF')

10

oct(x : int or x.__index__())

-> converts to octal str, prefix ‘0o’

- f'{10:#o}', f'{10:o}'-> ('0o12', '12')

Common Sequence Operations

Also supports - len O(1), min O(n), max O(n)

Membership

1

obj in seq

-> True if an item of seq is equal to x, else False

- Containment Test: contains(seq, obj)

- subsequence test for str, bytes, bytearray

- Time Complexity: O(n)

2

obj not in seq

-> False if an item of seq is equal to x, else True

- not_(contains(seq, obj))

- Time Complexity: O(n)

3

seq.count(x)

-> total number of occurrences of x in seq

- Time Complexity: O(n)

Concatenation

4

seq1 + seq2

-> concatenation of seq1 and seq2

- concat(seq1, seq2)

- Use str.join(), io.StringIO, bytes.join() or io.BytesIO for linear time complexity

- T&S for mutable: sum of len of sequences

- T&S for immutable: O(seq1*seq2)

5

seq * n

n * seq

-> equivalent to concatenating seq to itself n times

- items in the sequence seq are not copied; they are referenced multiple times (same objs ref n times)
- better init option: [[] for i in range(n)]

- T&S for mutable: O(len(seq) * n)

- T&S for immutable: O(len(seq) * n2)

Indexing

6

seq[i]

-> indexing, getitem(obj, k)

- neg index is looked up from the end
- Time Complexity: O(1)

7

seq.index(x[, i[, j]])

-> index of the first occurrence of x in seq (at or after index i and before index j)

- equivalent to s[i:j].index(x) but constant space

- Time Complexity: O(n)

Slicing

8

seq[i:j]

-> sequence of items with index k such that i <= k < j, shallow copy
- neg index is looked up from the end

- if i or j is greater than len(s), use len(s)
- if i is omitted or None, use 0
- if j is omitted or None, use len(s)

- if i is greater than or equal to j, the slice is empty

- getitem(seq, slice(i, j))

- Time Complexity: O(j-i) ~ O(n)

9

seq[i:j:k]

-> slice of seq from i to j with step k

- Ele x : x = i + n*k such that 0 <= n < (j-i)/k

- j is not included
- neg index is looked up from the end
- When k is positive, i and j are reduced to len(s) if they are greater. When k is negative, i and j are reduced to len(s) - 1 if they are greater.

- if i or j are omitted or None, they become “end” values (which end depends on the sign of k)

- k cannot be zero. If k is None, it is treated like 1
- Time Complexity: O((j-i)/k) ~ O(n)

Mutable Sequence Operations

- applicable to: list, bytearray

- t is any iterable

Add

1

seq.append(x)

- appends x to the end of the sequence
- AS: seq[len(seq):len(seq)] = [x]

- TC: O(1)

2

seq.insert(i, x)

- inserts x into seq at index i

- AS: seq[i:i] = [x]

- TC: O(n)

3

seq.extend(t)

seq += t

- extends seq with the contents of t

- TC: O(len(t))

4

seq *= n

- updates seq with its contents repeated n times

- items in the sequence are not copied; they are referenced multiple times

- TC: O(len(seq) * n)

Delete

5

del seq[i:j]

- removes i to j-1
- AS: s[i:j] = []
- TC: O(n)

6

del seq[i:j:k]

- removes the elements of s[i:j:k]
- TC: O(n)

7

seq.pop([i])

-> item at index i is removed and returned

- i is -1 if omitted hence last item is removed
- TC: O(1)

8

seq.remove(x)

- removes first occurrence of x from seq

- ValueError if x not found
- TC: O(n)

9

seq.clear()

- removes all items
- AS: del s[:]

- TC: O(1)

Modify

10

seqs[i] = x

- item at i is replaced by x

- TC: O(1)

11

seq[i:j] = t

- slice from i to j is replaced by iterable
- TC: O(n + len(t))

12

seq[i:j:k] = t

- elements of s[i:j:k] are replaced by those of t

- TC: O(n + len(t))

13

seq.reverse()

- reverses the items of seq in place

- TC: O(n)

Copy

13

seq.copy()
seq[:]

- creates a shallow copy of s

- TC: O(n)

constructor

1

namedtuple(typename, field_names, *, rename=False, defaults=None, module=None)

-> new tuple subclass named typename

- field_names are a sequence of strings such as ['x', 'y'], Alternatively, field_names can be a single string with each fieldname separated by whitespace and/or commas, for example 'x y' or 'x, y'

- Any valid Python identifier may be used for a fieldname except for names starting with an underscore

- If rename is true, invalid fieldnames are automatically replaced with positional names. For example, ['abc', 'def', 'ghi', 'abc'] is converted to ['abc', '_1', 'ghi', '_3']

- defaults can be None or an iterable of default values. For example, if the fieldnames are ['x', 'y', 'z'] and the defaults are (1, 2), then x will be a required argument, y will default to 1, and z will default to 2.

- If module is defined, the __module__ attribute of the named tuple is set to that value

5

6

Splitting

1

s.join(iterable)

-> concatenation of the strings in iterable

- TypeError will be raised if there are any non-string values in iterable

2

s.splitlines([keepends]

-> list of the lines in the string

- Line breaks are not included in the resulting list unless keepends is given and true.

See line break chars: https://docs.python.org/3/library/stdtypes.html#str.splitlines 

3

s.split(sep=None, maxsplit=-1)

-> list of the words in the string, using sep as the delimiter string

- If maxsplit is given, at most maxsplit splits are done (thus, the list will have at most maxsplit+1 elements) else no limit

- If sep is given, consecutive delimiters are not grouped together and are deemed to delimit empty strings

‘1,,2'.split(',') -> ['1', '', '2']) same for whitespace

4

s.rsplit(sep, maxsplit)

-> If maxsplit is given, at most maxsplit splits are done, the rightmost ones

- Except for splitting from the right, rsplit() behaves like split()

5

s.partition(sep)

-> Split the string at the first occurrence of sep, and return a 3-tuple containing the part before the separator, the separator itself, and the part after the separator

- If the separator is not found, return a 3-tuple containing the string itself, followed by two empty strings.

6

s.rpartition(sep)

-> Split the string at the last occurrence of sep

- like as s.partition(sep)

collections.deque

- Properties: unordered, unindexed, distinct, hashable objects

- Deques are a generalization of stacks and queues

- thread-safe, memory efficient appends and pops from either side

- If maxlen is not specified or is None, deques may grow to an arbitrary length. Otherwise, the deque is bounded to the specified maximum length. Once a bounded length deque is full, when new items are added, a corresponding number of items are discarded from the opposite end. (Bounded length deques provide functionality similar to the tail)

- very useful use cases: https://docs.python.org/3/library/collections.html#deque-recipes

- Implementation: A deque (double-ended queue) is represented internally as a doubly linked list.

Mutable Set Operations

- Since sets only define partial ordering (subset relationships), the output of the list.sort() method is undefined for lists of sets

- Binary operations that mix set instances with frozenset return the type of the first operand. For example: frozenset('ab') | set('bc') returns an instance of frozenset.

- The subset and equality comparisons do not generalize to a total ordering function. For example, any two nonempty disjoint sets are not equal and are not subsets of each other, so all of the following return False: a<b, a==b, or a>b

Mapping Types — dict

- maps hashable values to arbitrary objects

- dictionary is a mutable object

- dictionaries preserve insertion order. Note that updating a key does not affect the order. Keys added after deletion are inserted at the end (3.7)

- dictionaries compare equal if and only if they have the same (key, value) pairs (regardless of ordering). Order comparisons (‘<’, ‘<=’, ‘>=’, ‘>’) raise TypeError.

- 4 options as below

1. If no positional argument is given, an empty dictionary is created

2. if a positional argument is given and it is a mapping object, a dictionary is created with the same key-value pairs as the mapping object

3. If iterable gives, Each item in the iterable must itself be an iterable with exactly two objects. The first object of each item becomes a key in the new dictionary, and the second object the corresponding value.

- Elements are stored as dictionary keys and their counts are stored as dictionary values

bisect — Array bisection algorithm

- maintaining a list in sorted order without having to sort the list after each insertion

- see examples: https://docs.python.org/3/library/bisect.html#searching-sorted-lists

-> Locate the insertion point for x in a to maintain sorted order

- The parameters lo and hi may be used to specify a subset of the list which should be considered

- If x is already present in a, the insertion point will be before (to the left of) any existing entries

- return value is suitable for use as the first parameter to list.insert(index, x)

- TC: O(log(n))

2

bisect.bisect(a, x, lo=0, hi=len(a))

- returns an insertion point which comes after (to the right of) any existing entries of x in a

- LHS: all(val <= x for val in a[lo:i])

- RHS: all(val > x for val in a[i:hi])

- TC: O(log(n))

heapq — Heap queue algorithm

- aka: priority queue algorithm

- Min heaps are binary trees for which every parent node has a value less than or equal to any of its children

- heap[k] <= heap[2*k+1] & heap[k] <= heap[2*k+2] for all k

- smallest element is always the root, heap[0]

- todo read about heapsort, pq example, notes: https://docs.python.org/3/library/heapq.html#basic-examples
